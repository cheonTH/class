깃허브에 내 자료를 올리는 과정
1. 깃허브에 레포지토리를 만듦(비어있음)
2. 깃허브에 올리려는 폴더를 git이 관리하도록 설정
-> git init
3. 디렉토리의 모든 파일을 git의 스테이지 영역에 추가
-> git add . (디렉토리내의 변경된 모든 파일을 추가)

스테이징 영역
커밋을 하기 전에 임시로 올리는 공간

4. git에 커밋을 함
-> git commit -m "메시지"
여기까지는 깃허브에 올리는게 아님

5. 원격 저장소(github)와 연결
-> git remote add origin 저장소명
origin은 원격 저장소의 별칭 보통은 origin을 사용함

5. 브랜치 이름 변경
-> git branch -M main

6. 깃허브에 올림
-> git push origin main

checkout을 하는 이유
github레포지토리에 올린 코드를 실행환경에 다운로드하는 역할
github에 있는 소스코드를 현재 실행중인 가상머신(ubuntu)에 복사해오는 작업

모든 소스코드가 복사됨

CI / CD 스크립트를 작성할 때 리눅스 환경을 사용한다면 리눅스 명령어에 해서 숙지하고 있어야함

파일 / 디렉토리 관련 명령어
Is		현재 디렉토리 내용보기
cd[폴더명]		디렉토리 이동(cd ..은 상위 폴더로 이동)
pwd			현재 경로 출력
mkdir			새 디렉토리 만들기
cp (-r) [원본][복사본]	파일/폴더 복사
mv [원본][목적지]		이동 또는 이름변경
rm (-r)[파일]		파일 삭제(-r 폴더 삭제)

rm -r hello.text -> 파일이어도 상관은 없음
rm hello -> hello가 폴더면 에러가 발생

권한/소유권 관련
chmod [옵션] [권한] [파일 또는 폴더]

chmode +x hello.sh -> hello.sh파일에 실행 권한을 줌
